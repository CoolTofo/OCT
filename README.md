# OCT AI 创作工作室使用说明

OCT 是一个本地运行的 AI 创作工具。它把图片生成、视频生成、无限画布、ComfyUI 工作流、RunningHub 工作流、提示词模板和素材管理放在一个页面里，适合用来做短视频分镜、角色图、产品图、漫画素材、工作流测试和批量创作。

这份说明尽量说人话。你不需要懂代码，按步骤来就能用。

## 1. 一句话说明

打开 `启动服务.bat`，浏览器会自动进入 OCT。先去 `API 设置` 填好自己的 Key，再回到首页选择要用的功能。

常用入口：

- `视频生成`：用 Seedance / 即梦 / 兼容视频模型生成视频。
- `无限画布`：把图片、提示词、生成节点、视频节点、ComfyUI 节点放在一张大画布里串起来用。
- `API 设置`：配置各个平台的 API Key、模型名、接口地址。
- `ComfyUI 设置`：配置本地 ComfyUI 后端和导入工作流。
- `RunningHub`：配置 RunningHub 工作流，让画布里可以直接调用。
- `模板库`：保存常用提示词，不用每次重写。

## 2. 启动方法

在项目文件夹里双击：

```text
启动服务.bat
```

正常情况会弹出一个黑色窗口，并显示类似：

```text
Starting OCT Studio...
Visit: http://127.0.0.1:3010/
LAN: http://你的局域网IP:3010/
Press Ctrl+C to stop.
```

浏览器会自动打开：

```text
http://127.0.0.1:3010/
```

如果没有自动打开，就复制上面这个地址到浏览器。

不要关闭黑色窗口。黑色窗口就是本地服务，关掉以后网页就不能用了。

## 3. 第一次使用先做什么

第一次打开后，建议按这个顺序来：

1. 打开左侧或底部的 `API 设置`。
2. 添加你要用的平台，比如 ModelScope、APIMart、OpenAI 兼容平台、Seedance、RunningHub。
3. 填 API Key。
4. 填模型名。
5. 点击保存。
6. 回到 `视频生成`、`在线生图` 或 `无限画布` 开始用。

如果你只想用本地 ComfyUI，可以不用先配在线 API，但要去 `ComfyUI 设置` 填本地 ComfyUI 地址。

## 4. API 设置怎么填

`API 设置` 是整个工具的钥匙管理页面。哪里需要联网生成，基本都从这里读取配置。

你通常要填这些东西：

- `平台名称`：自己看得懂就行，比如 `我的 ModelScope`、`APIMart`、`Seedance`。
- `API Key`：平台给你的密钥。
- `Base URL`：接口地址。不同平台不一样。
- `协议`：一般选 OpenAI 兼容；RunningHub 就选 RunningHub。
- `生图模型`：给图片生成用。
- `LLM 模型`：给提示词润色、脚本拆分、对话等功能用。
- `视频模型`：给视频生成用。

简单理解：

- 想生成图片，就至少配置一个生图模型。
- 想生成视频，就至少配置一个视频模型。
- 想让画布里 AI 帮你写提示词，就至少配置一个 LLM 模型。

保存后，画布和各个功能页会自动读取这些模型。

## 5. 首页功能说明

首页是工具的总入口。

常见按钮大概是这些：

- `视频生成`：单独生成视频，适合快速测试提示词。
- `AI 网站`：放一些常用 AI 网站入口。
- `无限画布`：最重要的创作工作台。
- `API 设置`：管理 API Key 和模型。
- `ComfyUI 设置`：管理本地 ComfyUI 工作流。
- `RunningHub`：管理 RunningHub 工作流。

如果你不知道从哪开始，先进 `无限画布`。

## 6. 无限画布怎么用

无限画布可以理解成一张很大的桌面。你可以把图片、提示词、视频生成、ComfyUI 工作流都摆在上面，然后用连线把它们串起来。

常用操作：

- 拖动画布空白处：移动视野。
- 鼠标滚轮：缩放画布。
- 拖入图片：生成图片节点。
- 新建提示词节点：写文字。
- 新建生成节点：调用在线生图模型。
- 新建视频节点：调用视频模型。
- 新建 ComfyUI 节点：调用本地 ComfyUI 工作流。
- 新建 RunningHub 节点：调用 RunningHub 工作流。
- 从节点端口拖线：把上一个节点的图片或文字传给下一个节点。

最常见的用法：

1. 把参考图拖进画布。
2. 新建一个提示词节点，写你想要的画面。
3. 新建一个生图节点。
4. 把参考图和提示词连到生图节点。
5. 点生成。

生成出来的图片可以继续连到视频节点、ComfyUI 节点或新的生图节点。

## 7. 画布里常用节点

### 图片节点

用来放参考图、生成图、素材图。

你可以：

- 直接拖图片进画布。
- 从生成结果保存成节点。
- 把图片节点连到生图、视频、ComfyUI、RunningHub 节点。

### 提示词节点

用来写画面描述、镜头描述、角色描述。

建议写得具体一点，比如：

```text
一个年轻女性角色，站在雨夜街头，霓虹灯反光，电影感，浅景深，蓝紫色调
```

比这种更好用：

```text
美女，下雨，电影感
```

### 在线生图节点

用 API 设置里的生图模型生成图片。

适合：角色图、场景图、产品图、分镜图、参考图二次创作。

### 视频生成节点

用视频模型生成视频。

建议视频提示词写清楚：主体是谁、场景在哪里、镜头怎么动、动作怎么发生、风格是什么、时长和节奏是什么。

### ComfyUI 节点

调用你本机的 ComfyUI 工作流。适合本地模型生成、换脸、修复、放大、姿态控制和自己搭好的复杂工作流。

使用前要先去 `ComfyUI 设置` 配好工作流。

### RunningHub 节点

调用 RunningHub 平台上的工作流。适合不想在本地跑 ComfyUI，或者想把云端固定流程做成按钮式调用。

使用前要先在 `API 设置` 添加 RunningHub，再去 `RunningHub` 页面配置工作流。

## 8. ComfyUI 设置怎么用

如果你本机有 ComfyUI，默认地址通常是：

```text
127.0.0.1:8188
```

在 `ComfyUI 设置` 里可以做这些事：

- 添加或修改 ComfyUI 后端地址。
- 导入 ComfyUI 工作流。
- 把完整 workflow 转成可调用的 API 工作流。
- 选择哪些参数要在 OCT 画布里显示。
- 测试工作流是否能跑通。

简单流程：

1. 先启动你自己的 ComfyUI。
2. 打开 OCT 的 `ComfyUI 设置`。
3. 确认后端地址是 `127.0.0.1:8188`。
4. 导入或转换工作流。
5. 只保留画布里需要调的参数，比如提示词、图片、宽高、步数。
6. 保存。
7. 回到无限画布，新增 `ComfyUI 生成` 节点使用。

如果工作流参数太多，不要全暴露出来。只留下平时会改的字段，画布会更清爽。

## 9. RunningHub 怎么用

RunningHub 是云端工作流。你需要先有 RunningHub API Key。

使用步骤：

1. 打开 `API 设置`。
2. 点击添加 RunningHub。
3. 填 API Key。
4. 保存。
5. 打开 `RunningHub` 页面。
6. 添加 workflowId 和节点参数。
7. 保存工作流。
8. 回到无限画布，添加 RunningHub 节点。

简单理解：RunningHub 页面负责把云端工作流登记进 OCT，画布负责真正调用它。

## 10. 视频生成怎么写提示词

视频提示词不要只写画面，还要写动作和镜头。

不好用的写法：

```text
一个女孩在街上
```

更好用的写法：

```text
夜晚的城市街道，一个穿黑色风衣的女孩从霓虹灯下慢慢走过，雨水落在地面形成反光。镜头从侧后方平稳跟拍，人物回头看向镜头，情绪冷静神秘，电影感，浅景深，光影强烈。
```

可以按这个模板写：

```text
场景：在哪里
主体：谁在画面里
动作：正在做什么
镜头：推近、拉远、跟拍、环绕、俯拍
风格：电影感、写实、动画、广告片、短剧
细节：光线、天气、服装、表情、速度
```

## 11. 模板库怎么用

模板库用来保存常用提示词。

适合保存：人像风格模板、产品图模板、短剧分镜模板、视频镜头模板、常用负面词、固定品牌风格。

在无限画布里可以打开模板库，把模板直接放到画布里，或者填入当前选中的生成节点。

建议把模板名字写清楚，比如：

```text
短剧雨夜追逐镜头
产品白底电商图
小红书封面插画风
角色三视图提示词
```

以后找起来会快很多。

## 12. 文件都保存在哪里

常见目录：

- `output`：生成结果输出。
- `assets`：上传和素材相关文件。
- `data`：画布、模板、配置等数据。
- `workflows/custom`：自定义工作流。
- `data/comfyui_exports`：ComfyUI 导入和导出的工作流数据。
- `API/.env`：本地 API 配置文件。
- `history.json`：历史记录。

如果要把项目发给别人，注意不要把自己的 API Key 一起发出去。

## 13. 换电脑怎么用

最省事的方法：

1. 把整个 OCT 文件夹复制到新电脑。
2. 双击 `启动服务.bat`。
3. 如果提示缺依赖，再双击 `安装依赖.bat`。
4. 打开 `API 设置`，检查 API Key 和模型是否还在。
5. 如果使用 ComfyUI，检查 ComfyUI 后端地址是否正确。

如果 `python` 文件夹丢了，可以解压 `python.zip` 恢复内置 Python。

## 14. 上传到 GitLab

项目里有：

```text
上传到GitLab.bat
```

作用是把本地代码提交并上传到 GitLab。

使用前确认：

- 你在项目根目录运行。
- 当前网络能访问 GitLab。
- GitLab 账号有这个仓库的权限。
- 不要把 API Key、输出图、临时文件一起上传。

如果只是正常使用工具，不需要每天上传 GitLab。只有改了代码、工作流模板或要备份版本时才需要上传。

## 15. 更新 GitLab 最新版

项目里还有：

```text
更新GitLab最新版.bat
```

作用是把 GitLab 上的最新代码拉到本地。

更新前建议：

- 先确认自己有没有重要的本地改动。
- 如果刚改过代码，先上传或备份。
- 如果不确定，先复制一份整个 OCT 文件夹再更新。

## 16. 常见问题

### 页面打不开

先看黑色窗口还在不在。如果黑色窗口关了，重新双击 `启动服务.bat`。

如果窗口还在，浏览器打开：

```text
http://127.0.0.1:3010/
```

### 提示端口被占用

说明 3010 端口已经有一个 OCT 在运行，或者被别的软件占了。可以先关闭旧的黑色窗口，再重新启动。

### 报 No module named fastapi

说明 Python 依赖没装好。

处理方法：

1. 双击 `安装依赖.bat`。
2. 安装完成后重新双击 `启动服务.bat`。

### 生成失败

先按这个顺序查：

1. API Key 有没有填错。
2. 模型名有没有填错。
3. 平台余额或权限是否正常。
4. 网络能不能访问对应平台。
5. 提示词里有没有平台不允许的内容。
6. 如果是 ComfyUI，确认 ComfyUI 本身已经启动。

### 画布里没有模型可选

去 `API 设置` 检查：有没有添加平台、有没有填写模型名、平台有没有启用、保存按钮有没有点。

保存后回到画布刷新一下。

### ComfyUI 节点跑不动

检查这些：ComfyUI 是否已经启动、后端地址是不是 `127.0.0.1:8188`、工作流是否是 API 可运行格式、工作流需要的模型文件本地有没有、节点参数有没有填完整。

### RunningHub 节点失败

检查这些：RunningHub API Key 是否正确、workflowId 是否正确、工作流需要的节点参数是否配置完整、RunningHub 账号额度是否正常。

### 上传 GitLab 失败

常见原因：网络连不上 GitLab、没有登录 GitLab、当前目录不是 Git 仓库、分支不对、有冲突需要先处理。

如果你只是想用工具，GitLab 上传失败不影响本地生成。

## 17. 推荐工作流

### 做一张角色图

1. 打开无限画布。
2. 新建提示词节点，写角色外貌和风格。
3. 新建在线生图节点。
4. 连接提示词到生图节点。
5. 选择模型，点击生成。
6. 把满意的图保存到素材库。

### 用参考图做视频

1. 把参考图拖到无限画布。
2. 新建视频生成节点。
3. 写清楚动作和镜头。
4. 把图片连到视频节点。
5. 点击生成。

### 用 ComfyUI 工作流修图

1. 先启动 ComfyUI。
2. 在 OCT 的 `ComfyUI 设置` 导入工作流。
3. 保存需要调的参数。
4. 回到无限画布。
5. 把图片连到 ComfyUI 节点。
6. 调参数并运行。

### 做短剧分镜

1. 在画布里放剧本文字或提示词。
2. 用模板库套分镜模板。
3. 生成每个镜头的图。
4. 把满意的图连到视频节点。
5. 逐段生成视频。

## 18. 使用建议

- 提示词不要怕长，越具体越稳。
- 重要角色要固定描述，不要每次换说法。
- 画布用久了记得给画布起名字。
- 常用提示词放进模板库。
- 大工作流先小图测试，确认没问题再开高分辨率。
- 生成失败先看配置，不要急着改代码。
- API Key 不要截图发给别人。

## 19. 最短上手路线

如果你只想马上跑起来：

1. 双击 `启动服务.bat`。
2. 打开 `API 设置`。
3. 填一个能生图的平台和模型。
4. 保存。
5. 打开 `无限画布`。
6. 新建提示词节点。
7. 新建生图节点。
8. 连线。
9. 点击生成。

能跑通这一步，后面的 ComfyUI、RunningHub、视频生成都是在这个基础上继续加功能。
